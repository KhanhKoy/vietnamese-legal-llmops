5) Kiến thức mới đã tìm hiểu:
- `chunking.py` dùng để tách văn bản luật thành các đoạn nhỏ trước khi tạo embedding.
- Ưu tiên tách theo heading pháp lý như: Chương, Điều, Mục, Khoản.
- Nếu văn bản không có cấu trúc rõ ràng thì dùng cách chia theo cửa sổ ký tự (`chunk_size_chars`) và chồng lấn (`chunk_overlap_chars`).
- Mỗi chunk cần giữ lại metadata của document gốc để truy xuất nguồn sau này.
- Chunk nên có các trường: `chunk_id`, `document_id`, `text`, `chunk_index`, `metadata`.
- Cần chuẩn hóa text trước khi chia: bỏ `\r\n`, gộp khoảng trắng, giảm dòng trống thừa.
- Mục tiêu của chunking là giúp truy xuất chính xác hơn khi làm RAG cho dữ liệu pháp luật.

15) Cập nhật mới về xử lý tràn RAM:
- `src/rag_core/dataset_reader.py`
  - đọc metadata bằng streaming
  - đọc `content.parquet` theo batch nhỏ bằng `iter_batches()`
  - không dùng `pandas` cho toàn bộ row group lớn nữa
- `src/rag_core/pipeline.py`
  - xử lý theo luồng: document -> chunk -> embed -> add vào vector store
  - gom chunk theo batch nhỏ trước khi embed
  - tránh giữ toàn bộ document/chunk trong RAM cùng lúc
- `src/rag_core/vector_store.py`
  - lưu vector store xuống SQLite thay vì giữ toàn bộ embeddings trong RAM
  - đọc/chấm điểm theo từng batch khi search
  - giảm nguy cơ tràn RAM khi corpus lớn

16) Cách chạy hiện tại:
- chạy thử nhỏ trước:
  - `python scripts/build_index.py --metadata-limit 100 --content-limit 100`
- nếu ổn mới tăng dần limit

17) Ghi chú:
- mục tiêu chính là giảm RAM khi build index từ Hugging Face dataset lớn
- nếu vẫn còn tràn RAM, cần giảm thêm:

  - `embedding_batch_size`
- khi build xong, chatbot runtime chỉ cần load vector store và truy xuất, không cần chunk lại nếu dữ liệu không đổi

18) Cập nhật tối ưu RAM khi build index:
- `src/rag_core/dataset_reader.py`
  - đã thay cách giữ toàn bộ metadata trong RAM
  - metadata được stream vào SQLite tạm trên ổ đĩa
  - content vẫn đọc theo `iter_batches(batch_size=64)`
  - lookup metadata bằng SQLite thay vì Python dict lớn
  - giảm nguy cơ OOM khi metadata rất lớn

- `src/rag_core/vector_store.py`
  - thêm `commit()` để commit SQLite định kỳ
  - giảm tích tụ WAL cache khi `add()` liên tục
  - hỗ trợ pipeline flush/commit theo lô

- `src/rag_core/pipeline.py`
  - thêm `gc.collect()` sau mỗi lần commit định kỳ
  - commit sau mỗi `commit_interval` chunk
  - giảm bộ nhớ tạm của chunk dicts, embeddings buffer, document strings
  - build index theo luồng streaming, không giữ toàn bộ dữ liệu trong RAM

19) 3 vấn đề tràn RAM đã được xử lý:
- metadata lớn không còn load vào Python heap
- WAL cache SQLite được commit định kỳ
- đối tượng tạm được thu hồi bằng garbage collection

20) Cách build index hiện tại:
- chạy `scripts/build_index.py`
- build theo batch nhỏ
- commit định kỳ mỗi 500 chunk
- sau khi xong gọi `store.save()` và `gc.collect()`
